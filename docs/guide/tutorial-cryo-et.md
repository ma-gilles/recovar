# Tutorial: Cryo-ET on the CZII Phantom Dataset

This tutorial documents the CryoET Data Portal to RELION5 to RECOVAR path that was validated on March 25, 2026 for portal dataset `DS-10468`.

The workflow is:

1. Query the portal metadata and generate RELION5 STAR files plus a download manifest.
2. Download the aligned tilt stacks from the public portal bucket with `rclone`.
3. Run `relion_tomo_subtomo --stack2d` to extract 2D subtomogram particles.
4. Convert the RELION5 tomography output with `recovar parse_relion5_tomo`.
5. Run `recovar pipeline --tilt-series --tilt-series-ctf relion5`.
6. Run `recovar analyze` after the pipeline finishes.

!!! note "What RECOVAR expects"
    RECOVAR does not start directly from raw CryoET Data Portal annotations.

    The validated entry point is:

    - `tilt_series/aligned_tilt_series.star`
    - `Extract/.../particles.star` or `Refine3D/.../run_data.star`

    The generated `full_picks.star` file is a RELION5 particle-pick list for extraction. It is not yet a RECOVAR-ready particle STAR.

## Dataset Summary

- Dataset: `DS-10468`
- Portal title: `CZII 2-protein phantom (version 1.0) with defocus-based imaging`
- Object used here: `cytosolic ribosome`
- Tilt-series pixel size: `1.54 A`
- Tomogram voxel size in the portal annotations: about `5.006 A`

On March 25, 2026, the full ribosome manifest generated by the workflow below contained:

- `196` usable runs
- `12,498` portal particle annotations
- `387,438` particle-tilt images before RELION extraction
- `407,753,658,368` bytes of aligned tilt stacks to download

After `relion_tomo_subtomo`, RELION removed `34` edge particles and wrote `12,464` extracted particles. `recovar parse_relion5_tomo` then wrote `378,589` 2D tilt rows.

## Prerequisites

- RECOVAR installed in the repo `pixi` environment
- RELION5 available on `PATH`
- `rclone` available on `PATH`
- A separate helper Python environment with:
  - `cryoet-data-portal`
  - `scipy`
- A GPU for the RECOVAR stage

Install RECOVAR as usual:

```bash
pixi install
pixi run install-recovar
pixi run smoke-import-recovar
```

Create a small helper environment for the portal manifest script:

```bash
python -m venv .venv-portal
source .venv-portal/bin/activate
pip install cryoet-data-portal scipy
```

## 1. Generate RELION5 Metadata and a Download Manifest

From the repo root, run:

```bash
source .venv-portal/bin/activate
python scripts/cryoet/generate_relion5_ds10468_manifest.py \
    --output-root data/10468_ribosome_all
```

This writes:

- `data/10468_ribosome_all/tilt_series/aligned_tilt_series.star`
- one per-run tilt-series STAR under `data/10468_ribosome_all/tilt_series/`
- `data/10468_ribosome_all/full_picks.star`
- `data/10468_ribosome_all/download_manifest.json`
- `data/10468_ribosome_all/download_manifest.tsv`
- `data/10468_ribosome_all/summary.json`

If you want a smaller first pass, keep the same manifest generation step and then subset the resulting `download_manifest.tsv`, `aligned_tilt_series.star`, and `full_picks.star` before download / extraction.

## 2. Download the Aligned Tilt Stacks

The reliable download path for this dataset was public S3 via `rclone`. Direct HTTPS downloads repeatedly hit midstream read failures during validation.

```bash
export DOWNLOAD_JOBS=4
export LOG_DIR="$PWD/data/10468_ribosome_all/logs/download_rclone"
export RCLONE_RETRIES=10
export RCLONE_LOW_LEVEL_RETRIES=20

bash scripts/cryoet/download_manifest_rclone.sh \
    data/10468_ribosome_all/download_manifest.tsv
```

The downloader:

- maps each portal HTTPS URL to the public S3 bucket
- writes one per-run `rclone` log under `logs/download_rclone/`
- skips files that already match the expected byte size
- saves the aligned tilt stacks to `data/10468_ribosome_all/tiltseries/*.mrcs`

!!! note "Why `.mrcs`?"
    The portal aligned stacks are stored as `.mrc`, but RELION is happy to read the same data from a `.mrcs` path. The manifest generator writes `.mrcs` output paths on purpose.

## 3. Run RELION5 Extraction

Use the generated pick list and tilt-series STAR with `relion_tomo_subtomo`:

```bash
relion_tomo_subtomo \
    --p data/10468_ribosome_all/full_picks.star \
    --t data/10468_ribosome_all/tilt_series/aligned_tilt_series.star \
    --o data/10468_ribosome_all/extract/subtomo \
    --b 256 \
    --crop 192 \
    --stack2d \
    --j 8
```

Validated observations from the full run:

- RELION printed `Slash found in first particle name: not creating subdirectories for each tomogram`
- that warning was harmless here
- RELION removed `34` particles because they were too close to the edge in all images

The key output for RECOVAR is:

- `data/10468_ribosome_all/extract/subtomo/particles.star`

## 4. Convert RELION5 Tomography Output to RECOVAR's 2D Tilt Format

Run `parse_relion5_tomo` explicitly. This is the path that was validated during the full dataset run.

```bash
AGENT_ID=ds10468_parse
export SCRATCH_ROOT="/scratch/<user>"
export TMPDIR="$SCRATCH_ROOT/tmp/$AGENT_ID"
export PIXI_HOME="$SCRATCH_ROOT/pixi_home/$AGENT_ID"
export RATTLER_CACHE_DIR="$SCRATCH_ROOT/rattler_cache/$AGENT_ID"
mkdir -p "$TMPDIR" "$PIXI_HOME" "$RATTLER_CACHE_DIR"

unset PYTHONPATH PYTHONHOME CONDA_PREFIX VIRTUAL_ENV
export PYTHONNOUSERSITE=1

PIXI_PY="$(pixi run which python)"
"$PIXI_PY" -c "import pathlib,recovar,jax; repo=pathlib.Path.cwd().resolve(); assert str(pathlib.Path(recovar.__file__).resolve()).startswith(str(repo)+'/'); assert '.pixi/envs/default/' in str(pathlib.Path(jax.__file__).resolve())"

pixi run recovar parse_relion5_tomo \
    --tomograms data/10468_ribosome_all/tilt_series/aligned_tilt_series.star \
    --particles data/10468_ribosome_all/extract/subtomo/particles.star \
    --output data/10468_ribosome_all/particles_2d.star
```

On the validated full run, this loaded `196` tomograms, `12,464` particles, and wrote `378,589` 2D rows.

## 5. Run RECOVAR

Run the pipeline on the parsed 2D tilt STAR:

```bash
AGENT_ID=ds10468_pipeline
export SCRATCH_ROOT="/scratch/<user>"
export TMPDIR="$SCRATCH_ROOT/tmp/$AGENT_ID"
export PIXI_HOME="$SCRATCH_ROOT/pixi_home/$AGENT_ID"
export RATTLER_CACHE_DIR="$SCRATCH_ROOT/rattler_cache/$AGENT_ID"
mkdir -p "$TMPDIR" "$PIXI_HOME" "$RATTLER_CACHE_DIR"

unset PYTHONPATH PYTHONHOME CONDA_PREFIX VIRTUAL_ENV
export PYTHONNOUSERSITE=1
export XLA_PYTHON_CLIENT_PREALLOCATE=false

PIXI_PY="$(pixi run which python)"
"$PIXI_PY" -c "import pathlib,recovar,jax; repo=pathlib.Path.cwd().resolve(); assert str(pathlib.Path(recovar.__file__).resolve()).startswith(str(repo)+'/'); assert '.pixi/envs/default/' in str(pathlib.Path(jax.__file__).resolve())"

pixi run recovar pipeline \
    data/10468_ribosome_all/particles_2d.star \
    --mask sphere \
    --tilt-series \
    --tilt-series-ctf relion5 \
    --downsample 64 \
    --lazy \
    -o data/10468_ribosome_all/pipeline_out
```

Replace `SCRATCH_ROOT` with a fast local scratch path on your machine or cluster.

Validated observations from the full run:

- downsampling from `192` to `64` wrote a `6.20 GB` stack under `pipeline_out/downsampled/`
- downsampling throughput was about `845 images/sec`
- the first easy sanity-check output is `pipeline_out/output/volumes/mean_filt.mrc`
- later stages spend long periods in covariance / PCA with sparse log updates

## 6. Analyze the Heterogeneity

After the pipeline finishes, run:

```bash
pixi run recovar analyze \
    data/10468_ribosome_all/pipeline_out \
    --zdim=10 \
    --n-clusters=8 \
    --n-trajectories=2
```

Start with a modest `zdim` and cluster count for the first pass. Increase them once the mean, eigenvalue spectrum, and first cluster-center volumes look reasonable.

## Dataset-Specific Gotchas

These points mattered in the validated run:

- Portal annotations are mixed:
  - some runs provide `OrientedPoint`
  - some provide only `Point`
  - the manifest generator keeps both and sets Euler angles to `0,0,0` when orientation is absent
- RELION5 tilt-series STAR parsing required `_rlnCtfScalefactor`
  - the generated per-run tilt STARs write `_rlnCtfScalefactor = 1.0`
- Do not use the raw tomogram voxel counts for RELION centered coordinates
  - portal coordinates live on the tomogram voxel grid at about `5.006 A`
  - RELION `rlnCoordinate*` must be written in tilt-series pixels at `1.54 A`
  - the generator converts coordinates with `coord_scale = tomo_voxel_size / tilt_pixel_size`
  - the centered-coordinate origin uses alignment `volume_dimension / tilt_series_pixel_size`
  - for this dataset that gave effective sizes of about `4096 x 4096 x 1599`, not the raw `1260 x 1260 x 492`
- The validated rotation conversion was:
  - `R.from_matrix(rot).inv().as_euler("ZYZ", degrees=True)`
- Large tomography runs can look quiet for a long time
  - repeated variance-estimation logs are expected
  - some output files can appear before `run.log` visibly advances

## What to Inspect

Early in the run, inspect:

- `data/10468_ribosome_all/pipeline_out/output/volumes/mean_filt.mrc`
- `data/10468_ribosome_all/pipeline_out/output/volumes/mask.mrc`
- `data/10468_ribosome_all/pipeline_out/run.log`

After `recovar analyze`, inspect the usual RECOVAR outputs:

- eigenvalue plots
- PCA / UMAP cluster visualizations
- `analysis_10/kmeans/center*.mrc`

## Comparison to the SPA Tutorials

Compared with the [SPA worked examples](tutorial.md), the RECOVAR stages are familiar:

- `recovar pipeline`
- `recovar analyze`
- inspect the mean, eigenvalues, latent-space plots, and cluster-center volumes

The cryo-ET-specific work is entirely upstream:

- generating RELION5 tomography metadata from the portal
- downloading the aligned tilt stacks
- running RELION5 extraction
- converting the extracted RELION5 particles with `parse_relion5_tomo`
