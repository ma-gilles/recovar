# Student Runbook: Full-Atom Spike RECOVAR Experiments On Della

This is the fresh-clone path for reproducing the full-atom spike dataset-size
experiments on the Princeton Della cluster. It assumes the student has normal
cluster access, but is not running from `mg6942`'s home or scratch directories.

The workflow below does four things:

1. Clones this branch and builds RECOVAR in a clean pixi environment.
2. Runs a tiny smoke experiment first.
3. Runs the dataset-size sweep with saved kernel-regression candidates.
4. Postprocesses RECOVAR and, optionally, runs ModelAngelo scoring.

Run all commands from a login node unless a command explicitly uses `sbatch`.
Do not use conda.

## 0. Pick A Scratch Root

Choose one writable root for the whole experiment. The examples use the shared
group scratch area and a user-specific subdirectory.

```bash
export RECOVAR_STUDENT_ROOT=/scratch/gpfs/CRYOEM/gilleslab/tmp/$USER/spike_fullatom_student_$(date +%Y%m%d)
mkdir -p "$RECOVAR_STUDENT_ROOT"/{clone,slurmo,tmp,pixi_home,rattler_cache,inputs}
```

Everything generated by the commands below should land under
`$RECOVAR_STUDENT_ROOT`, except for read-only shared inputs.

## 1. Fresh Clone

```bash
cd "$RECOVAR_STUDENT_ROOT/clone"
git clone git@github.com:ma-gilles/recovar.git
cd recovar
git checkout codex/kernel-bandwidth-student-clean
git log -1 --oneline
```

If SSH access to GitHub is not set up, use the HTTPS URL instead:

```bash
git clone https://github.com/ma-gilles/recovar.git
```

## 2. Build The Environment

Use isolated pixi/cache directories. This avoids accidentally importing a
package from another checkout or from a user site-packages directory.

```bash
cd "$RECOVAR_STUDENT_ROOT/clone/recovar"

export AGENT_ID="student_setup_$(date +%Y%m%d_%H%M%S)"
export TMPDIR="$RECOVAR_STUDENT_ROOT/tmp/$AGENT_ID"
export PIXI_HOME="$RECOVAR_STUDENT_ROOT/pixi_home/$AGENT_ID"
export RATTLER_CACHE_DIR="$RECOVAR_STUDENT_ROOT/rattler_cache/$AGENT_ID"
mkdir -p "$TMPDIR" "$PIXI_HOME" "$RATTLER_CACHE_DIR"

unset PYTHONPATH PYTHONHOME CONDA_PREFIX VIRTUAL_ENV
export PYTHONNOUSERSITE=1

pixi install
pixi run install-recovar

PIXI_PY="$(pixi run which python)"
"$PIXI_PY" -m pip uninstall -y recovar || true
"$PIXI_PY" -m pip install -e . --no-deps --no-build-isolation --ignore-installed

module load cudatoolkit/12.8 || true
PYTHON="$PIXI_PY" make -C recovar/cuda clean all
module unload cudatoolkit/12.8 || true
```

Run the import/provenance checks:

```bash
pixi run smoke-import-recovar

"$PIXI_PY" - <<'PY'
import pathlib
import jax
import recovar

repo = pathlib.Path.cwd().resolve()
recovar_file = pathlib.Path(recovar.__file__).resolve()
jax_file = pathlib.Path(jax.__file__).resolve()
print("repo", repo)
print("recovar", recovar_file)
print("jax", jax_file)
assert str(recovar_file).startswith(str(repo) + "/")
assert ".pixi/envs/default/" in str(jax_file)
PY
```

If either assertion fails, stop and fix the environment before submitting jobs.

If any path is not readable, copy the missing inputs to a readable location and

ModelAngelo also needs a FASTA. If a shared FASTA is readable, use it:

```bash
export FASTA=/scratch/gpfs/GILLES/mg6942/runs/spike_full_atom_morph_validation_20260516/spike_P0DTC2_2P_ectodomain_27_1147_trimer_ABC.fasta
ls -lh "$FASTA"
```

If that path is not readable, derive a FASTA from the PDB stack:

```bash
export FASTA="$RECOVAR_STUDENT_ROOT/inputs/spike_from_morph050.fasta"
python - "$PDB_DIR/morph_050.pdb" "$FASTA" <<'PY'
from pathlib import Path
import sys

three_to_one = {
    "ALA": "A", "ARG": "R", "ASN": "N", "ASP": "D", "CYS": "C",
    "GLN": "Q", "GLU": "E", "GLY": "G", "HIS": "H", "ILE": "I",
    "LEU": "L", "LYS": "K", "MET": "M", "PHE": "F", "PRO": "P",
    "SER": "S", "THR": "T", "TRP": "W", "TYR": "Y", "VAL": "V",
}

pdb = Path(sys.argv[1])
out = Path(sys.argv[2])
chains = {}
seen = set()
for line in pdb.read_text().splitlines():
    if not line.startswith("ATOM"):
        continue
    chain = line[21].strip() or "A"
    resseq = line[22:26].strip()
    icode = line[26].strip()
    resname = line[17:20].strip()
    key = (chain, resseq, icode)
    if key in seen:
        continue
    seen.add(key)
    chains.setdefault(chain, []).append(three_to_one.get(resname, "X"))

out.parent.mkdir(parents=True, exist_ok=True)
with out.open("w") as f:
    for chain, seq in sorted(chains.items()):
        f.write(f">morph050_chain_{chain}\n")
        text = "".join(seq)
        for i in range(0, len(text), 80):
            f.write(text[i:i + 80] + "\n")
print(out)
PY
```

## 4. Quick GPU Sanity Check

Run this before submitting long jobs:

```bash
nvidia-smi || true
pixi run python - <<'PY'
import jax
print(jax.devices())
PY
```

On a login node it is normal not to see a GPU. The sbatch scripts repeat this
check on GPU nodes and abort if JAX sees no GPU.

## 5. Run A 10k Smoke Experiment

Submit one small array task first. This exercises volume rendering, simulation,
pipeline, compute_state, and saved candidate estimates.

```bash
cd "$RECOVAR_STUDENT_ROOT/clone/recovar"

export SMOKE_ROOT="$RECOVAR_STUDENT_ROOT/spike_smoke_noise100_b80"
export SMOKE_SHARED="$RECOVAR_STUDENT_ROOT/spike_smoke_noise100_b80_shared"

sbatch \
  --array=0-0 \
  --output="$RECOVAR_STUDENT_ROOT/slurmo/%x-%A_%a.out" \
  --error="$RECOVAR_STUDENT_ROOT/slurmo/%x-%A_%a.err" \
  --export=ALL,WORKDIR="$PWD",SCRATCH_ROOT="$RECOVAR_STUDENT_ROOT",BASE_ROOT="$SMOKE_ROOT",SHARED_ROOT="$SMOKE_SHARED",PDB_DIR="$PDB_DIR",MASK="$DEFAULT_MASK",N_IMAGES_VALUES_STR="10000",NOISE_LEVEL=100.0,RENDER_BFACTOR=80 \
  scripts/experiments/spike_fullatom_state_sweeps/submit_fullatom_noise100_b80_dataset_size.sbatch
```

Monitor it:

```bash
tail -f "$RECOVAR_STUDENT_ROOT"/slurmo/spike-fa-n100-b80-*.out
```

## 6. Postprocess The Smoke Experiment

```bash

  --output="$RECOVAR_STUDENT_ROOT/slurmo/%x-%j.out" \
  --error="$RECOVAR_STUDENT_ROOT/slurmo/%x-%j.err" \
  --export=ALL,WORKDIR="$PWD",SCRATCH_ROOT="$RECOVAR_STUDENT_ROOT",ROOT="$SMOKE_ROOT",LABEL="student_smoke_noise100_b80",TARGET_STATE=50,MASK="$DEFAULT_MASK" \
  scripts/experiments/spike_fullatom_state_sweeps/postprocess_fullatom_dataset_size.sbatch
```

Important outputs:

```text
$SMOKE_ROOT/plots/
$SMOKE_ROOT/mean_subtracted_metrics_20260518_r12/
```

The standard FSC plot compares the unfiltered compute_state map to the target
GT volume inside the supplied mask using FSC 0.5. The mean-subtracted metrics
compare `(estimate - mean_gt)` to `(target_gt - mean_gt)`.

## 7. Run The Dataset-Size Sweep

After the 10k smoke run succeeds, run the actual sweep. This example uses the
latest noise 100, render B80 settings:

```bash
export FULL_ROOT="$RECOVAR_STUDENT_ROOT/spike_fullatom_consistency_grid256_noise100_b80"
export FULL_SHARED="$RECOVAR_STUDENT_ROOT/spike_fullatom_consistency_grid256_noise100_b80_shared"
export N_IMAGES_VALUES_STR="10000 30000 100000 300000 1000000"

sbatch \
  --array=0-4%3 \
  --output="$RECOVAR_STUDENT_ROOT/slurmo/%x-%A_%a.out" \
  --error="$RECOVAR_STUDENT_ROOT/slurmo/%x-%A_%a.err" \
  --export=ALL,WORKDIR="$PWD",SCRATCH_ROOT="$RECOVAR_STUDENT_ROOT",BASE_ROOT="$FULL_ROOT",SHARED_ROOT="$FULL_SHARED",PDB_DIR="$PDB_DIR",MASK="$DEFAULT_MASK",N_IMAGES_VALUES_STR="$N_IMAGES_VALUES_STR",NOISE_LEVEL=100.0,RENDER_BFACTOR=80 \
  scripts/experiments/spike_fullatom_state_sweeps/submit_fullatom_noise100_b80_dataset_size.sbatch
```

To run the easier noise 30 version, change only these variables:

```bash
export FULL_ROOT="$RECOVAR_STUDENT_ROOT/spike_fullatom_consistency_grid256_noise30_b80"
export FULL_SHARED="$RECOVAR_STUDENT_ROOT/spike_fullatom_consistency_grid256_noise30_b80_shared"
```

For 3M images, include it in the list and extend the array:


testing file staging and queue behavior.

## 9. Reuse A Finished Dataset For Another Target State

The state-reuse script does not regenerate particles or rerun the pipeline. It
symlinks `03_dataset`, `04_ground_truth`, `05_masks`, and `06_pipeline` from a
finished source sweep, then runs only `compute_state` at a new target latent
point.

State 25:

```bash
export STATE25_ROOT="$RECOVAR_STUDENT_ROOT/spike_fullatom_state0025_reuse_noise100_b80"

sbatch \
  --array=0-4%3 \
  --output="$RECOVAR_STUDENT_ROOT/slurmo/%x-%A_%a.out" \

  --export=ALL,WORKDIR="$PWD",SCRATCH_ROOT="$RECOVAR_STUDENT_ROOT",SOURCE_ROOT="$FULL_ROOT",STATE_ROOT="$STATE25_ROOT",TARGET_STATE=25,N_IMAGES_VALUES_STR="$N_IMAGES_VALUES_STR" \
  scripts/experiments/spike_fullatom_state_sweeps/submit_state25_compute_state_reuse.sbatch
```

```bash
export STATE0_ROOT="$RECOVAR_STUDENT_ROOT/spike_fullatom_state0000_reuse_noise100_b80"
```

## 10. Optional: ModelAngelo Scoring

ModelAngelo runs through the RELION 5 module on A100 nodes. It expects a TSV
with one row per dataset:

```bash
export MA_ROOT="$FULL_ROOT/modelangelo_fixed_b_m100"
mkdir -p "$MA_ROOT"

for n in 10000 30000 100000 300000 1000000; do
  nlabel="$(printf 'n%08d' "$n")"
  printf '%s\t%s/%s/runs/%s_seed0000\n' "$nlabel" "$FULL_ROOT" "$nlabel" "$nlabel"
done > "$MA_ROOT/runs_all.tsv"

wc -l "$MA_ROOT/runs_all.tsv"
cat "$MA_ROOT/runs_all.tsv"
```

Submit ModelAngelo:

```bash
sbatch \
  --array=0-4%2 \
  --output="$RECOVAR_STUDENT_ROOT/slurmo/%x-%A_%a.out" \
  --error="$RECOVAR_STUDENT_ROOT/slurmo/%x-%A_%a.err" \
  --export=ALL,SCRATCH_ROOT="$RECOVAR_STUDENT_ROOT",ROOT="$MA_ROOT",RUN_LIST="$MA_ROOT/runs_all.tsv",FASTA="$FASTA",ADHOC_BFAC=-80,ANGPIX=1.25 \
  scripts/experiments/modelangelo_spike_scoring/submit_modelangelo_fixed_b_dataset_size.sbatch
```

Use `ADHOC_BFAC=-100` for B100-rendered simulations and `ADHOC_BFAC=-80` for
B80-rendered simulations.

Score ModelAngelo outputs:

```bash
sbatch \
  --output="$RECOVAR_STUDENT_ROOT/slurmo/%x-%j.out" \
  --error="$RECOVAR_STUDENT_ROOT/slurmo/%x-%j.err" \
  --export=ALL,WORKDIR="$PWD",ROOT="$MA_ROOT",REFERENCE_PDB="$PDB_DIR/morph_050.pdb",RESIDUE_RANGES="$RESIDUE_RANGES" \
  scripts/experiments/modelangelo_spike_scoring/score_modelangelo_dataset_size.sbatch
```

Key outputs:

```text
$MA_ROOT/postprocess/
$MA_ROOT/modelangelo/
$MA_ROOT/moving_region_all_atom_same_element_rmsd.csv
$MA_ROOT/plots/
```

For state 25 or state 0, use the corresponding state root, reference PDB, and
residue range file. The PDB morphs are one-indexed: state 0 corresponds to
`morph_001.pdb`, state 25 to `morph_026.pdb`, and state 50 to `morph_050.pdb`
for the historical scoring used here.

## 13. Common Failures

`JAX did not see a GPU`

The job landed somewhere without a visible GPU or the environment was
contaminated. Check the Slurm partition, `nvidia-smi`, and the pixi provenance
messages in the log.

`recovar.__file__` is outside this clone

The editable install is stale or another package is shadowing the clone. Rerun
the pixi setup in section 2 with `PYTHONNOUSERSITE=1` and no conda variables.

`recovar/cuda/libcuda_backproject.so` is missing

The sbatch scripts try to build it. If that fails, manually run:

```bash
module load cudatoolkit/12.8 || true
PYTHON="$(pixi run which python)" make -C recovar/cuda clean all
module unload cudatoolkit/12.8 || true
```

Missing PDB, mask, FASTA, or residue ranges

Do not edit the scripts. Copy the input to a readable directory and pass
`PDB_DIR=...`, `MASK=...`, `FASTA=...`, or `RESIDUE_RANGES=...` through
`--export=ALL,...`.

Slurm cannot write logs

Always pass `--output="$RECOVAR_STUDENT_ROOT/slurmo/%x-%A_%a.out"` and the
matching `--error` path in submit commands.

Low-count ModelAngelo tasks fail

That can happen. Use the scoring job anyway; it records failed or empty models
as rows with parse errors instead of failing the whole summary.
